UI RULES — SAPIRA

This is a controlled system, not a playground.

------------------------------------------------------------

TYPOGRAPHY

- Text must NEVER feel accidental
- Line breaks must feel intentional
- Avoid widows/orphans
- Avoid awkward wrapping

------------------------------------------------------------

MOBILE TYPOGRAPHY

- Respect container width
- No overflow
- No forced nowrap
- Let text wrap naturally

------------------------------------------------------------

SPACING

- Always from scale
- No arbitrary values
- Vertical rhythm must be consistent

------------------------------------------------------------

BUTTONS

Primary:
- solid
- calm hover

Secondary:
- subtle background (rgba)
- no strong borders

------------------------------------------------------------

HOVER

- subtle only
- opacity or slight shift
- no movement > 4px

------------------------------------------------------------

SHADOWS

- layered
- soft
- realistic

Bad shadow:
→ hard edge
→ flat blur
→ centered

Good shadow:
→ directional
→ diffused
→ subtle gradient

------------------------------------------------------------

LAYERS

- avoid stacking chaos
- no accidental overlap
- no clipping

------------------------------------------------------------

ERROR PREVENTION

Always check:
- overflow-x
- wrapping
- mobile breakpoints
- stacking context
